### 2026年2月27日：修复 OpenAI OGL 验证 unknown_error
Model:GPT-5.2
AI_Tools:Codex App
- Codex OAuth 默认 redirect_uri 调整为 `http://localhost:1455/auth/callback`，避免非 loopback 回调导致 OpenAI 验证失败。
- Core 增加本地回调监听（`127.0.0.1:1455` / `[::1]:1455`），收到回调后自动调用模块 `/auth/oauth/exchange` 完成 Token 获取。
- 模块侧 `/auth/oauth/start` 仅在 `redirect_uri` 一致时复用 pending flow，避免旧 flow 继续返回错误的回调地址。

### 2026年2月27日 21:53：修复 Provider 保存与导入稳定性问题
Model:GPT-5.2
AI_Tools:Codex App
- 修复 Providers 页面在“编辑 Provider”时无法保存：由于 Step1 的隐藏必填字段 `accountName` 仍触发表单原生校验，导致 submit 不触发；进入 Step2 时禁用该字段并移除 required，返回 Step1 时恢复。

- 模块 Provider 自动注册逻辑优化：`internal/modules/manager.go` 注册 provider alias 时，若已存在同名 module provider，保留用户的 Enabled/ConfigJSON，仅刷新 Endpoint/Protocol（端口可能变更）；若存在同名非 module provider 则不覆盖。
- 默认模型初始化逻辑优化：仅在模型不存在时写入默认模型，避免每次重启覆盖用户对默认模型的启用状态调整。
