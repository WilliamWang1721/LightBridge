LightBridge 使用指南与配置说明

快速开始

LightBridge 是一个轻量级 AI 模型网关，提供 OpenAI 兼容的 API 接口，支持多种后端 Provider 的统一接入与路由。

  • 默认监听地址：127.0.0.1:3210
  • OpenAI 兼容端点：/v1/*（如 /v1/chat/completions）
  • OpenAI 别名端点：/openai/v1/*
  • 模型变体语法：使用 model@providerAlias 指定特定 Provider

客户端配置

将您的 AI 客户端（如 Cherry Studio、Cursor 等）的 Base URL 指向本实例，并使用在「设置」页面生成的 Client API Key 进行认证。

Base URL:  http://127.0.0.1:3210/v1
API Key:   sk-lb-xxxxxxxxxxxxxxxx
如果您使用的客户端不支持自定义 Base URL，可以尝试使用 /openai/v1/* 别名端点。

Codex（OpenAI）配置指南

Codex 是 OpenAI 提供的编程辅助模型。通过 LightBridge 的 openai-codex-oauth 模块，您可以使用 OAuth 方式接入 Codex。

  • 前往「模块 Marketplace」安装 openai-codex-oauth 模块
  • 前往「提供商 Provider」页面，点击「添加」，选择「Codex（OpenAI）」
  • 默认使用「生成 OAuth 链接」进行登录授权
  • 点击生成 OAuth 链接后，会自动显示回调 URL 快速输入区（在高级选项外），可直接手动换取 Token
  • 若 OAuth 链接方式出现问题，请在 ChatGPT 设置中允许 Device Code 登录，然后使用「生成 Device Code」方式
  • 高级用户可展开「高级选项」手动导入 Token 或上传 Auth.json 文件（支持批量导入）

Provider 管理

Provider 是 LightBridge 连接后端 AI 服务的核心组件。每个 Provider 代表一个 API 端点。

  • 支持的协议：forward、anthropic、http_openai、http_rpc、grpc_chat、codex
  • Provider 类型：builtin（内置）或 module（模块提供）
  • 可在 Provider 列表页面进行启用/禁用、批量操作
  • 支持卡片视图、列表视图和模型视图三种显示模式

路由与模型映射

LightBridge 通过路由规则将客户端请求的模型名称映射到具体的 Provider 和上游模型。

  • 在「路由 Router」页面配置模型与 Provider 的映射关系
  • 支持多 Provider 负载均衡与故障转移
  • 使用 model@provider 语法可绕过路由直接指定 Provider

模块系统

LightBridge 支持通过模块扩展功能。模块可以提供新的 Provider 类型、认证方式或工具。

  • 在「模块 Marketplace」浏览和安装可用模块
  • 模块分为 Provider 模块、Auth 模块和 Tools 模块
  • 安装后可在模块详情中进行配置、启动/停止和卸载
  • 模块配置支持可视化编辑和 JSON 高级编辑两种模式
